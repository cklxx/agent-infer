## Experiment Option 1 — log counter(推荐,fastest,~10 LOC)

**Rationale**:在 production 路径插入 timing log,跑 fresh-server 第 1 个 cap=8 burst,
看哪个 phase 最慢。Direct evidence,no profiler setup。

**Insertion site**:`infer/src/scheduler/cuda/prefill.rs:659` 的 `forward_prefill_batch` 调用

**Patch**(非 commit,纯 experiment,run 后 revert):
```rust
// At infer/src/scheduler/cuda/prefill.rs:659
// BEFORE (existing):
let forward_result = self.model.forward_prefill_batch(
    &requests,
    &mut self.states,
    batch.uses_paged.then_some(&mut self.paged_kv_pool),
);

// AFTER (experiment):
let _t_pf = std::time::Instant::now();
let _bs = requests.len();
let forward_result = self.model.forward_prefill_batch(
    &requests,
    &mut self.states,
    batch.uses_paged.then_some(&mut self.paged_kv_pool),
);
let _ms = _t_pf.elapsed().as_secs_f64() * 1e3;
// First 20 requests post-server-start: log every prefill timing
// Use static AtomicUsize to avoid lock contention.
static EXPT_COUNT: std::sync::atomic::AtomicUsize =
    std::sync::atomic::AtomicUsize::new(0);
let n = EXPT_COUNT.fetch_add(1, std::sync::atomic::Ordering::Relaxed);
if n < 20 {
    log::info!(
        "EXPT prefill #{}: bs={} ms={:.2}",
        n, _bs, _ms,
    );
}
```

Same pattern at `infer/src/scheduler/cuda/decode.rs:281` for decode-side baseline。

**How to run**(in codex shell):
```fish

# 2. Rebuild

# 3. Start fresh server

    --model models/Qwen3-4B \

sleep 10

    --concurrent 8 --duration 30s

# 5. Inspect log

# 6. Kill server, revert patch

```

**Decision criteria(SOLID license-or-kill)**:
- **PASS prefill axis**:first prefill `ms=` ≥ 2× steady-state prefill `ms=`,且 decode 第 1 个 `ms=` 与 steady-state decode 接近 → prefill 是 cold-start 主因 → Phase 1 LOC 合理
- **KILL prefill axis**:first prefill 与 steady-state 接近,但 first decode ≥ 2× steady-state → decode 才是冷启动主因 → Phase 1 wrong axis,pivot
- **Mixed**:两个都慢 → 都是冷启动来源,prefill warmup 部分有效但不足 → Phase 1 ROI 减半
- **Neither**:两个都接近 steady-state → 33% degraded 来自 paged-KV alloc / L2 cold / 其他 → Phase 1 完全 wrong axis,KILL P0.3,投入 nsys trace 重新定位

## Experiment Option 2 — nsys 30s trace

**Rationale**:GPU-level evidence,看 NVTX/kernel 占比。

**Setup**:per `docs/resources/infer-cuda-profiling-wrappers.md`(nsys wrapper)。

**Run**:
```fish
nsys profile --output /tmp/p0.3-expt2 \
    --trace cuda,nvtx,osrt \
    --duration 30 \
    ./target/release/infer serve --model models/Qwen3-4B --num-slots 8 --port 18000 &
sleep 5
./scripts/bench_guidellm.sh p0.3-expt2 --concurrent 8 --duration 25s
wait
nsys stats /tmp/p0.3-expt2.nsys-rep | head -50
```

**Decision criteria**:
- **PASS prefill**:`prefill::*` NVTX window first-burst ≥ 2× steady-state,decode 接近 steady
- **KILL prefill**:其他 phases 占比更大

**Note per CLAUDE.md §0**:nsys per-NVTX-window framing 必须 cross-check
per-wall-clock framing(2026-05-08 EOD+19 framing trap)。看 first-burst
3 个 prefill 的 absolute ms 总和 / TTFT,不是 NVTX window 占比。

## Experiment Option 3 — dummy curl pre-warm A/B

**Rationale**:zero code change,纯 bench A/B,用 1 个 dummy 请求"预热"prefill 路径,看第 2 个 burst bimodal 是否消失。

**Run**:
```fish
# A: cold (no pre-warm)
./target/release/infer serve --model models/Qwen3-4B --num-slots 8 --port 18000 &
sleep 5
./scripts/bench_guidellm.sh p0.3-expt3-cold --concurrent 8 --duration 60s --runs 5
kill %1

# B: pre-warmed (1 dummy curl before bench)
./target/release/infer serve --model models/Qwen3-4B --num-slots 8 --port 18000 &
sleep 5
curl -s http://localhost:18000/v1/completions \
    -d '{"model":"qwen3","prompt":"hello","max_tokens":1}' > /dev/null
./scripts/bench_guidellm.sh p0.3-expt3-warm --concurrent 8 --duration 60s --runs 5
kill %1

# Compare bimodal distribution
diff <(jq .ttft.p99 p0.3-expt3-cold-r*.json) \
     <(jq .ttft.p99 p0.3-expt3-warm-r*.json)
```

**Decision criteria**:
- **PASS prefill**:warm A/B 没有 33% degraded path(bimodal collapses to unimodal)
- **KILL prefill**:bimodal 仍在,1-token dummy 不足以预热 cap=8 GEMM shapes → either need cap=8-shape pre-warm or wrong axis

**Note**:1-token dummy 只触发 prompt-length=1 prefill GEMM,可能不覆盖 cap=8 burst 的 prompt-length GEMM shapes。Better dummy:8 个 concurrent requests with 模拟 prompt 长度。
