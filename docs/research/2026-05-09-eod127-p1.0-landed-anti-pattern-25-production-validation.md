## Key insight for skill v1.8.0 enhancement

**Anti-pattern #25 sub-rule candidate**(refinement):
> "Implementation context must be strict subset of hypothesis target context
> at EVERY granularity:syntactic boundary,empirical boundary,phase boundary
> (e.g. decode-pure vs decode-batched vs prefill)。Each granularity may
> require its own gate。"

P1.0's final form has **3-layer gating**:
1. Outer:`is_hybrid_w4_marlin()` weight type check(syntactic)
2. Middle:`batch > 8 + env-on + aligned`(empirical boundary per R4 #6)
3. Inner:**explicit decode/prefill phase dispatch**(phase boundary,critical for hybrid)

Each layer prevents a different class of leak。Without explicit phase
dispatch(layer 3),mixed-batch contains decode rows leaking through W4A8
even when `batch > 8` is satisfied — this was the residual leak that
codex review caught post-first-run。
